# Что делается, как делается и почему

## 1. Почему local web + local API
Такой подход даёт быстрый и чистый UI, а потом его можно упаковать в desktop-shell без переписывания интерфейса.

## 2. Почему frontend и backend разделены
UI, runtime pipeline и будущая ML-интеграция развиваются с разной скоростью. Разделение снижает связанность и упрощает тестирование.

## 3. Почему основная единица анализа это flow/session
Текущий ML-контекст ближе к flow-level признакам, а не к анализу отдельных пакетов. Это снижает риск расхождения между обучением и runtime.

## 4. Почему сейчас mock pipeline
Модели и production capture ещё не финализированы. Mock-режим позволяет уже сейчас запускать интерфейс, API и историю без небезопасного низкоуровневого кода.

## 5. Почему хранение на JSON/NDJSON
Для первого каркаса этого достаточно. История уже может копиться по дням, а retention policy до 30 дней реализуется просто и прозрачно.

## 6. Почему контракты вынесены отдельно
Сериализуемый feature contract нужен для будущей совместимости между train-time preprocessing и inference-time preprocessing.
