# AnomalyNet IDS — Руководство по установке

## Требования

| | Windows (`install.bat`) | Windows (`install-windows.bat`) | Linux |
|---|---------|-------|-------|
| ОС | Windows 10/11 (x64) | Windows 10/11 (x64) | Ubuntu 22.04+, Debian, RHEL, Alt Linux, любой дистрибутив |
| Python | 3.10+ ([скачать](https://python.org)) | устанавливается автоматически | 3.10+ (`sudo apt install python3 python3-venv`) |
| Git | не нужен | устанавливается автоматически | обычно уже установлен |
| Node.js | **не нужен** (фронтенд уже собран) | устанавливается автоматически | **не нужен** |
| Права Admin | **не нужны** | нужны — UAC «Да», без пароля | нужны для Live-захвата трафика (`sudo`) |
| RAM | ≥ 2 ГБ | ≥ 2 ГБ | ≥ 2 ГБ |

## Установка

### Windows — `install.bat` (простой, без Admin)

Используйте если приложение уже скачано и распаковано. Установщик сделает всё автоматически:

1. Проверит Python 3.10+
2. Создаст виртуальное окружение `backend\.venv`
3. Установит зависимости (`pip install`)
4. Создаст ярлык **AnomalyNet IDS** на рабочем столе (с иконкой)
5. Добавит в **автозапуск Windows** (реестр `HKCU` — без прав администратора)
6. Запустит приложение — браузер откроется на `http://localhost:8000`

> **Пароль не нужен. Admin не нужен.** Двойной клик — и готово.

---

### Windows — `install-windows.bat` (полный, рекомендуется)

Полная установка с нуля: скачивает Python, Git, Npcap, клонирует репозиторий с GitHub, добавляет в **автозапуск через Task Scheduler**.

**Запуск:**  
Двойной клик на `install-windows.bat` → появится UAC-диалог → нажать **«Да»** → установщик сделает всё сам.

> **Пароль не нужен.** UAC показывает только кнопку «Да» — без ввода пароля.  
> Если аккаунт Windows является администратором (большинство домашних ПК) — пароль не спрашивается.

Что установщик делает:

1. Проверяет права администратора (UAC)
2. Устанавливает Python 3.11 (если не найден — через winget или прямая загрузка)
3. Устанавливает Git (если не найден)
4. Клонирует `AnomalyNet-gui` и `AnomalyNet-ml` из GitHub
5. Создаёт виртуальное окружение и устанавливает Python-зависимости
6. Собирает React-фронтенд (устанавливает Node.js если нужно)
7. Устанавливает Npcap для Live-режима
8. Добавляет в **автозапуск** через Task Scheduler (без пароля, через `InteractiveToken`)
9. Запускает приложение — браузер откроется на `http://localhost:8000`

После этой установки **обновление** доступно через вкладку About → «Применить обновление».

### Linux — `bash install.sh`

```bash
bash install.sh
```

Установщик сделает всё автоматически:

1. Проверит Python 3
2. Создаст виртуальное окружение `backend/.venv`
3. Установит зависимости
4. Создаст ярлык в меню приложений и на рабочем столе
5. Добавит в **автозапуск** (`~/.config/autostart` — без sudo)
6. Запустит приложение — браузер откроется на `http://localhost:8000`

> **sudo не нужен** для установки. Нужен только для Live-режима захвата трафика (см. ниже).

## Запуск после установки

После первой установки приложение запускается автоматически при входе в систему.

Для ручного запуска:

| Способ | Windows | Linux |
|--------|---------|-------|
| Ярлык | Двойной клик **AnomalyNet IDS** на рабочем столе | Двойной клик на `.desktop` ярлыке |
| Файл | Двойной клик **`launch.bat`** | `bash launch.sh` |
| Браузер | `http://localhost:8000` | `http://localhost:8000` |

Если сервер уже запущен — `launch.bat`/`launch.sh` просто откроет браузер, не создавая второй процесс.

## Режимы работы

По умолчанию приложение запускается в **Demo-режиме** — генерирует случайные события для демонстрации интерфейса. Реальный трафик не анализируется.

Для анализа реального трафика переключитесь в **Live-режим** в настройках (вкладка Settings).

### Windows Live-режим и Npcap

Для захвата трафика на Windows нужен **Npcap**:

1. Скачайте Npcap — [npcap.com](https://npcap.com/#download)
2. При установке выберите нужный вариант:

| Вариант установки Npcap | Запуск AnomalyNet | Нужен Admin? |
|------------------------|-------------------|-------------|
| **«Allow non-admin users to capture packets»** ✓ | Обычный запуск | **Нет** |
| Без этой галочки | Правой кнопкой → «Запуск от имени администратора» | Только кнопка «Да» |

> **Про «пароль администратора»:** если ваш аккаунт Windows является администратором (большинство домашних ПК) — UAC просто показывает диалог с кнопкой **«Да»**, никакого пароля вводить не нужно. Пароль спрашивается только если аккаунт не-администраторский.

### Linux Live-режим

```bash
# Вариант 1: запуск с sudo (разовый)
sudo bash launch.sh

# Вариант 2: дать права навсегда (рекомендуется, не нужен sudo при каждом запуске)
sudo setcap cap_net_raw,cap_net_admin=eip $(readlink -f $(which python3))
bash launch.sh   # дальше без sudo
```

## Решение проблем

### Windows: Python не найден
Скачайте Python 3.10+ с [python.org](https://python.org).  
При установке обязательно поставьте галочку **«Add Python to PATH»**.

### Windows: ошибка захвата трафика (требует администратора)
Установите Npcap с галочкой **«Allow non-admin users»**, либо запускайте `launch.bat` правой кнопкой → «Запуск от имени администратора».

### Windows: порт 8000 занят
Откройте `config/settings.json` и измените `"port": 8000` на другой порт.

### Windows: install-windows.bat запрашивает пароль Microsoft
Это происходит если в скрипте используется старый метод регистрации задачи.  
Скачайте актуальную версию — в ней Task Scheduler настраивается через `InteractiveToken` без ввода пароля.

### Linux: Permission denied при захвате трафика
```bash
sudo setcap cap_net_raw,cap_net_admin=eip $(readlink -f $(which python3))
```

### Linux: нет модуля venv
```bash
sudo apt install python3-venv   # Debian/Ubuntu
sudo dnf install python3        # Fedora
```

### Приложение не открывается в браузере
Перейдите вручную: [http://localhost:8000](http://localhost:8000)  
Убедитесь что в консоли нет ошибок и сервер запустился.
